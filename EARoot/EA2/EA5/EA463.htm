Note that in MODEM we need to be more precise about how an activity maps to a function. In previous versions of MODAF, if more than one Function realised an activity (or vice versa) we simply created more relationships. However, that led to ambiguity because in a trade-off of multiple SVs for an OV, the multiple relationships could be understood as options.<br /></p><p><br /></p><p>In MODEM, it is clear. If several activities are realised by one function, an ActivityGroup is created containing those activities and it is that group that maps to the function. Similarly for multiple functions realising one OperationaActivity, or many functions that together realise a group of OperationalActivities.<br /></p>
